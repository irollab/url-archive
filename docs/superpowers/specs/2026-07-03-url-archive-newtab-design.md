# URL Archive 新标签页设计文档

## 目标

URL Archive 新标签页是浏览器扩展的专属首页工作台。它替代浏览器默认新标签页，优先解决“收藏太多、找起来麻烦”的问题，同时展示插件的剪藏、导入、回访、AI 找回等核心价值。

首版目标不是做一个完整浏览器主页，而是让用户打开新标签页后，能在 1-3 秒内通过视觉书签、分类或搜索找到目标网页。

## 设计方向

### 产品定位

采用“浏览器首页工作台”方向：

- 首屏核心是视觉书签墙，满足快速打开常用网站。
- 左侧保留浏览器原书签分类层级，满足按分类找。
- 右侧展示 URL Archive 插件能力，包括今日回访、AI 找回、最近剪藏、剪藏/导入入口。
- 搜索同时覆盖浏览器书签和 URL Archive 剪藏索引。

### 视觉方向

默认采用“清爽生产力”风格：

- 背景克制、信息层级清楚，适合高频作为新标签页使用。
- 前卫感来自布局、卡片、图标、微交互和密度切换，而不是强烈装饰。
- 支持深色前卫模式作为可选主题，不作为默认。

## 信息架构

### 顶部区域

顶部承担全局操作：

- URL Archive 品牌标识。
- 全局搜索框。
- 视图密度切换：紧凑、标准、大图标。
- 剪藏当前页。
- 导入浏览器书签。
- 设置入口。

搜索框是主入口，默认执行本地关键词搜索。输入更自然语言化的内容时，提供 AI 找回入口或 AI 结果区。

### 左侧分类导航

左侧展示书签分类：

- 保留浏览器原始文件夹层级。
- 父级分类统计包含所有子分类书签数量。
- 点击分类后，中间书签墙显示该分类及子分类下的内容。
- 支持“未整理”“最近导入”“暂存队列”等虚拟分类。

### 中间视觉书签墙

中间是首屏重点：

- 每个书签以 favicon / 首字母占位图标 + 标题 + 域名 + 分类/标签展示。
- 支持书签和剪藏混排，但必须显示来源标识。
- 点击卡片打开网页。
- 卡片提供编辑入口，可编辑标题、分类、标签、备注。
- 视图密度影响卡片尺寸和每屏数量。

三档密度：

- 紧凑：更多书签，适合重度用户。
- 标准：默认，兼顾信息量和可读性。
- 大图标：更直观，适合常用入口。

### 右侧插件功能区

右侧展示插件能力：

- 今日回访：推荐长期未打开但可能有价值的收藏。
- AI 找回：自然语言找回入口。
- 最近剪藏：展示最近写入 Obsidian 的 URL Archive 笔记。
- 插件操作：剪藏当前页、导入书签、打开设置、同步/刷新索引。

响应式策略：

- 宽屏：右侧栏常驻。
- 中等宽度：可折叠。
- 窄屏：右侧功能区收进抽屉，只保留搜索、分类和书签墙。

## 搜索设计

采用双模式搜索。

### 本地关键词搜索

默认模式，必须稳定可用。搜索范围：

- 标题
- URL
- 域名
- 分类路径
- 标签
- 备注
- 摘要
- AI 关键词、别名、回访意图

结果按相关性排序，并显示命中的来源信息。

### AI 找回

AI 找回是增强能力，不阻塞本地搜索。

首版可以实现为：

- 搜索框旁边的“AI 找回”按钮。
- 或本地搜索结果下方的“用 AI 找回”结果区。

AI 找回应支持类似表达：

- “那个智能记账工具”
- “之前收藏的语义搜索方案”
- “排查数据库慢查询的文章”

如果未配置 BYOK，AI 找回入口应显示配置提示，不影响本地搜索。

## 编辑能力

新标签页支持编辑索引信息：

- 标题
- 分类路径
- 标签
- 备注

首版编辑只修改 URL Archive 插件索引，不直接修改浏览器原生书签。这样可以避免误改 Chrome/Edge 收藏夹，也符合当前“导入为索引”的设计。

后续可增加“同步回浏览器书签”的高级开关，但不纳入首版。

## 数据来源

### 浏览器书签

通过 `chrome.bookmarks.getTree()` 导入浏览器书签，转换为 `SavedClip` 索引：

- `source: bookmark`
- 保留原文件夹路径 `folder`
- 保存 favicon URL
- 保存标题、URL、域名、分类、标签、备注

导入不会写入 Obsidian，也不会修改浏览器原书签。

### 剪藏内容

剪藏仍写入 Obsidian vault 的 `URL Archive/` 目录，同时保存到插件本地索引：

- `source: clip`
- 包含 AI 摘要、标签、关键词、别名、回访意图
- 使用 canonical URL 去重，避免 Chrome/Edge 对同一 URL 写出多份文件

### 跨浏览器边界

Chrome 和 Microsoft Edge 的扩展本地存储天然不共享。首版不解决完整跨浏览器索引同步。

共享边界：

- Obsidian 是剪藏 `.md` 的共享来源。
- 浏览器书签导入索引仍各浏览器独立。

后续可以设计“从 Obsidian 反向重建索引”或“导出/导入 URL Archive 索引”来改善跨浏览器共享。

## 技术设计

### Manifest

使用 `chrome_url_overrides.newtab` 指向新标签页入口。

WXT 中新增 `entrypoints/newtab/`：

- `index.html`
- `main.ts`
- `style.css`

继续复用现有 background 消息和索引模块，不为新标签页另建独立数据源。

### 数据接口

复用或新增 background 消息：

- `SEARCH_CLIPS`
- `BOOKMARK_FOLDERS`
- `SAVED_CLIP_STATS`
- `IMPORT_BROWSER_BOOKMARKS`
- `OPEN_REVISIT`
- `UPDATE_SAVED_CLIP`
- 新增 `GET_DASHBOARD_DATA`：一次性返回统计、分类、默认书签列表、最近剪藏、回访推荐。

`GET_DASHBOARD_DATA` 用于减少新标签页启动时的多次消息往返。

### 前端组件边界

首版可继续使用原生 TypeScript，不引入 React/Vue，保持项目现有模式。

建议拆分为小模块：

- `newtab/state.ts`：当前筛选、搜索、密度、主题状态。
- `newtab/render.ts`：渲染函数。
- `newtab/actions.ts`：与 background 通信。
- `newtab/view-model.ts`：将 `SavedClip` 转换为卡片、分类、右侧栏数据。

如果新标签页继续膨胀，再考虑引入轻量组件框架。

## 首版范围

必须实现：

- 覆盖浏览器新标签页。
- 顶部搜索。
- 左侧分类导航。
- 中间视觉书签墙。
- favicon / 首字母占位。
- 紧凑 / 标准 / 大图标三档密度。
- 宽屏右侧栏、窄屏抽屉。
- 本地关键词搜索。
- 导入浏览器书签。
- 编辑标题、分类、标签、备注。
- 打开网页时记录回访。

首版增强但可降级：

- 深色前卫模式。
- 今日回访。
- 最近剪藏。
- AI 找回入口。

不纳入首版：

- 修改浏览器原生书签。
- 跨 Chrome/Edge 自动同步本地索引。
- 将导入书签批量写入 Obsidian。
- 复杂拖拽排序。
- 多设备云同步。

## 错误处理

- 无书签权限：提示需要重新授权或重新加载扩展。
- 未导入书签：显示空状态，引导点击“导入书签”。
- favicon 加载失败：显示域名首字母占位。
- Obsidian 不可用：剪藏走现有暂存队列，新标签页展示暂存状态。
- BYOK 未配置或 AI 请求失败：隐藏或降级 AI 找回，本地搜索继续可用。
- 索引为空：展示导入书签、剪藏当前页、打开设置三个入口。

## 测试策略

单元测试：

- 分类树和父级计数。
- 搜索过滤和排序。
- 密度设置持久化。
- 编辑后索引更新。
- dashboard view model 输出。

集成验证：

- `npm run compile`
- `npm test`
- `npm run build`

手工验证：

- Chrome/Edge 重新加载扩展后，新标签页被 URL Archive 覆盖。
- 导入书签后分类和书签墙出现。
- 搜索关键词能命中标题、分类、标签、备注。
- 编辑分类后左侧分类计数刷新。
- favicon 失败时显示占位。
- 宽屏显示右侧栏，窄屏右侧栏进入抽屉。

## 后续迭代

- 支持按访问频率、最近访问、AI 相关性排序。
